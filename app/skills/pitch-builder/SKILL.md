# Pitch Builder

## Objetivo
Construir um pitch de vendas estruturado e personalizado para a empresa-alvo, integrando dados coletados pelo briefing e perfil do interlocutor.

## Quando usar
- Vendedor pede "montar pitch", "preparar apresentação", "script de vendas"
- Após já ter coletado dados sobre a empresa e/ou interlocutor
- Quando quer consolidar toda a inteligência em um roteiro de reunião

## Workflow

1. **Revisar contexto**: verificar se já existe briefing de empresa e perfil de interlocutor no pitch atual
2. **Estruturar abertura**: primeiros 2 minutos — como criar conexão
3. **Diagnóstico**: perguntas para confirmar as dores identificadas
4. **Apresentação de valor**: conectar produto/serviço com as dores confirmadas
5. **Prova social**: cases similares ao perfil da empresa
6. **Tratamento de objeções**: antecipar 2-3 objeções prováveis com respostas prontas
7. **Fechamento**: próximo passo concreto

## Formato de saída

```
## Pitch — [EMPRESA]

### 1. Abertura (2 min)
- Quebra-gelo: [sugestão personalizada ao interlocutor]
- Contexto: "Vi que vocês [referência a notícia/dado recente]..."
- Objetivo da conversa: [1 frase]

### 2. Diagnóstico (5 min)
Perguntas para validar dores:
1. "Como vocês estão lidando com [dor 1]?"
2. "Qual o impacto de [dor 2] no dia a dia da equipe?"
3. "Vocês já tentaram resolver [dor 3] de alguma forma?"

### 3. Apresentação de Valor (10 min)
Para cada dor confirmada:
- **Dor:** [descrição]
- **Solução:** [como seu produto resolve]
- **Resultado:** [métrica ou benefício concreto]

### 4. Prova Social
- Case: [empresa similar] conseguiu [resultado] em [tempo]

### 5. Objeções Prováveis
| Objeção | Resposta |
|---------|----------|
| "Preço alto" | ... |
| "Já temos solução" | ... |
| "Preciso consultar" | ... |

### 6. Fechamento
- Próximo passo sugerido: [ação concreta com data]
- Frase de fechamento: "..."
```

## Regras
- O pitch deve soar como uma conversa, não um script rígido.
- As perguntas de diagnóstico são mais importantes que a apresentação — vendedor que pergunta vende mais.
- Cada bloco deve ser independente — o vendedor pode pular ou reordenar conforme a conversa flui.
- Se não houver dados suficientes sobre a empresa, alertar e sugerir pesquisar antes.
- Integrar dados do catálogo de produtos quando disponível.
